# CHANGELOG — CleanupTemp Pro

---

## [3.2.1] — 2026-03-01

### Добавлено

**Системный трей (Tray)**
- Иконка приложения живёт в трее на протяжении всей сессии — программа не занимает место на панели задач
- Контекстное меню трея: «Открыть», «Сканировать сейчас», «Выход»
- Строка статуса в меню трея обновляется в реальном времени: «Готов» / «Сканирование...» / «Очистка...» / «Очищено: 1.2 ГБ»
- Balloon-уведомление «Очистка завершена!» появляется только когда окно скрыто — не спамит поверх открытого UI
- Кнопка сворачивания и системное сворачивание (Win+D) скрывают окно в трей вместо минимизации на панель задач
- Двойной клик по иконке и клик левой кнопкой восстанавливают окно
- `ShutdownMode=OnExplicitShutdown` — процесс остаётся живым пока окно скрыто

**Круговая диаграмма очистки (PieChart)**
- После завершения очистки вкладка автоматически переключается на визуальный отчёт
- Пончиковая диаграмма показывает разбивку освобождённого места по категориям
- Staggered-анимация: сегменты появляются последовательно с задержкой 80 мс каждый
- Легенда справа: цветовая точка, название категории, процент, размер
- Категории меньше 1.5% от общего объёма автоматически объединяются в «Прочее»
- Кнопка «← К результатам» возвращает к списку файлов
- Tooltip на каждом сегменте: название + размер + процент

**Белый список исключений**
- Новая вкладка «🛡️ Исключения» — файлы и папки в этом списке никогда не удаляются
- Контекстное меню в списке найденных файлов: «Исключить из очистки» и «Исключить всю папку»
- Кнопка ручного добавления пути через диалог выбора файла
- Кнопка «Очистить всё» с подтверждением
- Хранение в `%AppData%\CleanupTempPro\exclusions.json`, загрузка при старте
- Проверка исключений во время сканирования: O(1) через `HashSet<string>`

**Анализ больших файлов**
- Новая вкладка «📦 Большие файлы» — поиск по всем подключённым дискам
- Порог размера: 100 МБ / 500 МБ / 1 ГБ / 2 ГБ / 5 ГБ (по умолчанию 500 МБ)
- Сканирование в фоновом потоке с батчевым обновлением UI (50 файлов за раз)
- Результаты отсортированы по убыванию размера
- Иконки по типу файла: 💿 образы, 🗜️ архивы, 🎬 видео, 🎵 аудио, ⚙️ установщики и др.
- Контекстное меню: «Открыть папку» (explorer /select) и «Удалить файл» с подтверждением
- Поддержка отмены через `CancellationToken`

### Изменено

- Метод `SwitchTab` переработан на enum `ActiveTab` вместо bool-флага — поддержка 4 вкладок
- Иконка трея загружается из ресурсов сборки (`pack://`) с fallback на папку рядом с `.exe`
- `App.xaml.cs` полностью переписан: инициализация окна перенесена в `OnStartup`, добавлен публичный API `UpdateTrayStatus` / `ShowBalloon`

### Инфраструктура

- `UseWindowsForms=true` добавлен в `.csproj` для `NotifyIcon` (единственный WinForms-класс в проекте)
- Добавлены `using`-алиасы во все `.cs`-файлы для разрешения конфликтов `System.Drawing` ↔ `System.Windows.Media` и `System.Windows.Forms` ↔ `System.Windows`: `Color`, `Point`, `Brush`, `Brushes`, `FontFamily`, `FontStyle`, `MouseEventArgs`, `Orientation`, `ColorConverter`, `HorizontalAlignment`, `VerticalAlignment`, `CheckBox`, `Application`
- Файлы затронуты: `MainWindow.xaml.cs`, `App.xaml.cs`, `AboutWindow.xaml.cs`, `CustomDialog.xaml.cs`

## [3.2.0] — 2026-02-27

### Исправлено

**Зависание при старте**
- WMI-запрос к Storage API (`MSFT_PhysicalDisk`) вынесен в `Task.Run` — окно открывается мгновенно
- Поля `ScanDegree` / `CleanDegree` теперь `volatile static` и обновляются после завершения фонового запроса
- Дефолт при недоступности WMI: 2 потока (компромисс между HDD-безопасностью и скоростью)

**Гарантированное восстановление служб Windows Update**
- Службы `wuauserv` и `BITS` теперь восстанавливаются в `finally`-блоке, обёртывающем все фазы очистки
- Восстановление происходит даже при исключении, отмене пользователем или аварийном завершении фазы
- Предыдущая логика с `try/catch` внутри фазы допускала выход без восстановления при `OperationCanceledException`
